Объектно-ориентированное программирование.

Современная разработка программного обеспечения ведётся с применением объектно-ориентированного программирования (ООП). Все популярные языки программирования поддерживают парадигму ООП.
У объектно-ориентированного программирования есть несколько важных преимуществ:
  • Разработка на основе объектов, описанных в классах, полностью соответствует инженерному подходу к задаче и системному мышлению.
  • ООП можно рассматривать как моделирование отношений между объектами реального мира с помощью "информационных двойников" этих объектов.
  • ООП помогает бороться со сложностью объёмных программных систем, поскольку с его помощью можно построить многоуровневую систему, в которой более высокий уровень скрывает сложность объектов на более ниизких уровнях.
  • Средства ООП позволяют удобно управлять зависимостями между программными компонентами.
Объектно-ориентированное программирование (object-oriented programming) основано на создании активных объектов, которые взаимодействуют друг с другом и таким образом выполняют дейставия в программе. Каждый объект можно охарактеризовать его состоянием (какие данные он хранит в себе) и поведением (как он умеет обрабатывать данные).
В объектно-ориентированной программе все действия выполняются объектами. Объекты создаются, чтобы выполнить действия по обработке данных, которые хранятся тоже в объектах. Для этого одни объекты обращаются к другим и "просят" их выполнить какие-то действия, и так — по цепочке. Инициаторами обработки могут быть объекты, которые взаимодействуют с человеком (через UI), реагируют на события, получают данные по сети, обрабатывают внешние сигналы.

Ключевые понятия ООП можно описать так:
  • Объект (object) — это индивидуальная программная сущность, которая относится к определённому классу. Объект ещё называют экземпляром (instance) какого-то класса. Объект в программе представляет нужный физический объект (человека, станок, подразделение) или программный компонент (окно, базу данных, сервер).
    Примеры объектов на web-странице: кнопка (button), список (select), поле ввода (textarea), изображение (image).
    Примеры объектов в жизни: работник (worker), книга (book), сервер (server).
     
  • Атрибут (attribute) — это внутренняя переменная (поле данных) объекта. При описании классе для каждого атрибута указывается его имя (и тип хранимых данных). Имена атрибутов (описанные в классе) у всех объектов класса одинаковые, а значения могут быть разными. Текущее состояние (данные) каждого объекта — это значения его атрибутов. Термины атрибут и свойство (property) могут применяться как синонимы.
    Примеры атрибутов: высота и ширина у изображения (image.height и image.width),  фамилия у работника (worker.family_name), заглавие у книги (book.title), адрес у сервера (server.address).
     
  • Метод (method) — это операция над объектом со своим именем и списком параметров.  Метод описывается в классе в виде внутренней функции (member function) или процедуры. Поведение объекта представлено набором его методов. Причём для "управления" объектом со стороны "посторонних" объектов других классов могут использоваться только общедоступные (public) методы.
    Примеры методов: "узнать возраст" работника (worker.age()), "назначить оклад" работнику (worker.change_salary(new_wage)), "изменить цену" у книги (book.price=new_price), "запросить" ресурс с сервера (response=server.get(request)).
     
  • Класс (class) — это описание общих характеристик всех однотипных объектов: их структуры (состава), атрибутов и методов. В описании класса указывается какие данные и в каком виде могут храниться в экземплярах класса (его свойства) и какие действия можно совершать над этими объектами (его методы).
    Примеры классов: Изображение (Image), Работник (Worker), Книга (Book), Сервер (Server).
     
  • Абстракция данных (data abstraction) означает вот что:
    1) в классах мы описываем только нужные нам (для решаемой задачи) свойства объектов и действия над ними, а от несущественных подробностей мы "абстрагируемся" (не рассматриваем);
    2) объекты одного уровня дают нам достаточно информации, чтобы решать задачу на этом уровне, а подробности и механизмы взаимодействия, лежащие на более низких уровнях нам для этого не требуются и мы можем не разбираться в них.
     
  • Инкапсуляция (encapsulation) — программный механизм, который позволяет объекту "спрятать в капсулу" (от посторонних объектов) свои данные и внутренние методы доступа к ним, разрешая изменять своё состояние только через открытые методы.
    Например, мы не разрешаем изменять у книги атрибут "цена", а предоставляем для этого метод price(), который выполняет все проверки нового значения, прежде, чем изменить цену.
     
  • Наследование (inheritance) — механизм "передачи по наследству" от "родительского класса" (parent class) к "дочернему" (child class) информации о структуре объектов (атрибутах) и операциях (методах изменения данных). То есть всё, что описано у "родителя", доступно и "детям", но "дети" добавляют собственные описания или могут заменять родительские определения.
    Например, для общего класса Shape (Фигура) можно описать подклассы: Circle (Окружность) и Polygon (Многоугольник), а для многоугольника — подклассы Triangle (Треугольник) и Square (Квадрат) со своими характеристиками.
     
  • Полиморфизм (polymorphism) — дословно "многие формы", а по сути "разные смыслы" у методов (действий над объектами) с одинаковыми именами.
    Например, операция "+" в выражении  n + 1  это сложение чисел,  в выражении  message + "!" это сцепление строк, а в конструкции  array1 + array2 это объединение списков.
    Ещё пример: все геометрические фигуры унаследовали от родительского класса (Shape) метод  show(), который показывает фигуру на экране, но каждый подкласс переорпеделил его так, чтобы правильно показывать каждую из фигур. Поэтому можно у любого "наследника" класса Shape можно вызывать метод, но будет выполняться метод конкретного класса.
Абстрактные классы.

Когда нужно описать общее понятие, для которого точно не будут создаваться объекты, но которая будет "родителем" для подклассов, то описыватся абстрактный класс, который помечается ключевым словом abstract.
Например, класа Shape (геометрическая фигура вообще) вполне может быть абстрактным.

Интерфейсы.

Ключевым словом interface описывается "контейнер" для действий, которые могут "наследоваться разными классами". Он очень похож на класс, но содержит только методы. А любой класс может заявить себя "наследником" такого интерфейса и реализовать эти методы. Тогда появится возможность применять одинаковые действия к объектам всех классов, которые реализовали этот интерфейс.
Например, интерфейс Sortable требует наличия в классе операции сравнения <=> (на меньше, равно или больше) и описывает метод sort(). Тогда объекты всех классов, которые унаследовали этот интерфейс (и изменили метод сортировки под особенности своих данных) можно упорядочить, вызывая метод сортировки (это также пример полиморфизма):

      workers.sort()        # работники будут сортироваться по ФИО
      books.sort()          # книги - по названию
      servers.sort()        # серверы - по IP-адресам
      colors.sort()         # цвета - по кодам RGB

Q&A.
Q: Чем класс отличается от типа данных ?
A: В каждом объектно-ориентированном языке программирования есть предопределённые классы. Много пользовательских классов описано в библиотеках языка, которые подключаются по мере необходимости. Программист может описывать свои классы, которые, по сути, являются пользовательскими типами данных.
В одних ОО-языках помимо классов есть типы данных, называемые "простыми", "примитивными" или "встроенными" (целые и дробные числа, строки, логические величины). В других языках (например в Ruby), все типы данных являются классами.
