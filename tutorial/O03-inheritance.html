Наследование.

В ООП есть очень мощный механизм — наследование (inheritance). С помощью наследования можно создавать новые классы на основе существующих. При этом новый (дочерний) класс "по наследству" получает все свойства и методы существующего (родительского) класса. Но в дочернем классе можно описывать дополнительные свойства и методы. А также переопределять методы родительского класса.

В этом примере на основе класса "Подразделение" (Department) описывается класс "Удалёное подразделение" (RemoteDivision): 
# Ruby:
class RemoteDivision < Department 
  def initialize(id, code, name, parent_id, remote_address_id)
    super(id, code, name, parent_id);
    @remote_address_id = remote_address_id
  end
end 

// JavaScript:
class RemoteDivision extends Department { 
  constructor(id, code, name, parent_id, remote_address_id) {
    super(id, code, name, parent_id);
    this.remote_address_id = remote_address_id;
  }
}
Наследование сокращает размер исходников, позволяет вносить изменения в единственном месте.
