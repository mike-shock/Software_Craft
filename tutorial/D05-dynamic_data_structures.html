Динамические структуры данных.

Иногда требуется использовать сложные структуры данных: деревья, графы, очереди. Обычно такие разветвлённые структуры создаются во время выполнения программы, поэтому они называются динамическими (dynamic data structure). Динамические структуры данных можно создавать с помощью ссылок (reference).

Дерево.

Примеры деревьев: эволюционное дерево видов, структура подразделений в организации. Или иерархия элементов в документе HTML:



Чтобы представить в программе дерево (tree), можно применить уже известную структуру данных. Например, дерево можно представить в виде хэша: 
// JavaScript:
var tree = {
  "html": {
    "head": {
      "style": {
      },
      "script": {
      }
    },
    "body": {
      "h1": {
      },
      "div": {
        "h2": {
        },
        "img": {
        },
        "span": {
        },
      }
    }
  }
};

Связь между классами типа "Наследование" (relationship) — ещё один пример дерева: 
Животные
-----Млекопитающие
     -----Приматы
          -----Люди
     -----Кошачьи
          -----Котики
-----Птицы

Другой пример — связь типа "Агрегация" (aggregation), когда в одном объекте-контейнере содержатся другие объекты-контейнеры, в которых находятся ещё объекты... 
Компьютер 
-----Корпус
     -----Материнская плата
          -----Процессор
          -----Память
     -----Жёсткий диск
-----Монитор
-----Клавиатура

Дерево в реляционной таблице можно сохранить, если иметь дополнительную колонку, в которой будет храниться ссылка на родительскую запись: внешний ключ (parent_id) на первичный ключ (id) в этой же таблице:

id   data parent_id
10 Корень дерева NULL
11 Узел на уровне 1 10
12 Узел на уровне 2 11
13 Узел на уровне 1 10
21 Узел на уровне 3 12

Граф.

Граф (graph) можно создать в программе, используя ссылки между объектами-узлами графа. Граф также можно представить в виде массива, например, матрицы смежности.
